🏠 US Household Income Analysis (SQL)

📌 Project Overview

This project focuses on cleaning and analyzing US household income data using SQL.
The objective was to improve data quality, validate records, and explore income trends across different states, counties, and household types.

The project follows a practical data analyst workflow:

Data Cleaning,
Data Validation,
Exploratory Data Analysis (EDA),
Insight Generation

🛠️ Tools & Technologies

MySQL,
SQL,
Data Cleaning,
Exploratory Data Analysis (EDA)

📂 Dataset

The dataset includes information related to:

State Names,
Counties,
Cities,
Household Income Statistics,
Land & Water Area Data,
Income Mean & Median Values,
Household Types

🧹 Data Cleaning Process

The data cleaning phase included:

Identifying and removing duplicate records.

Standardizing inconsistent state and type values.

Correcting missing or inaccurate location data.

Validating household income records.

Improving overall dataset consistency and reliability.

Key SQL Techniques Used:

ROW_NUMBER(),
PARTITION BY,
DELETE,
UPDATE,
JOIN,

📊 Exploratory Data Analysis

The analysis explored:

Income distribution across US states.

Average household income by region.

Differences between household types.

Land and water area comparisons.

Regional income trends and patterns.

Key SQL Concepts Used:

Aggregations (AVG, SUM),
Grouping & Filtering,
Joins,
Data Validation Queries,
Trend Analysis

🔍 Key Insights

Some states showed significantly higher average household income levels.

Regional differences revealed strong income variation across the US.

Certain household types had noticeably higher income averages.

Data inconsistencies can significantly affect analytical accuracy if not properly cleaned.
